아래 원문 기사와 재작성 기사를 함께 검수하고 반드시 JSON만 출력한다. 다른 텍스트는 금지한다.

[매체 특성] {media_tone}

즉시 탈락 조건
- 내부 추론 노출: Step, Phase, 생각, 검토, 분석 같은 사고과정 문구 포함
- 미완성 기사: 문장이나 단락이 중간에 끊김, 말줄임표로 종료, 결론 없이 종료
- 포맷 오염: 마크다운, HTML 태그 포함
- 사실 위반: 원문에 없는 수치, 고유명사, 사건을 사실처럼 단정
- 시제 승격: 계획, 검토, 예정, 추진 중을 시행, 완료, 확정으로 바꿈
- 원문 근거 없는 효과 과장: 제목, 리드문, 본문 첫머리에서 원문에 없는 수혜자 변화나 파급효과를 단정
- 홍보문체: 칭찬, 수상, 의지 표명, 다짐 위주로 흘러 실제 변화나 작동 구조가 보이지 않음
- 빈약한 효용: 발표 사실만 나열하고 누가 어떤 변화를 겪는지 설명하지 못함

채점 항목
- source_alignment: 제목과 리드문이 원문에 있는 조치와 근거에 밀착해 있는가
- source_alignment: 원문에 없는 해석을 사실처럼 승격하지 않았는가
- source_alignment: 원문이 직접 제시한 변화 동사와 결과를 불필요하게 추상 명사로 흐리지 않았는가
- lead: 첫 2문장 안에 핵심과 수혜자, 해결책, 임팩트가 있는가
- utility: 첫 2문단 안에 무엇이 달라졌는지와 왜 지금 중요한지가 드러나는가
- utility: 독자가 "그래서 무엇이 달라지나"라는 질문에 답을 얻는가
- added_value: 단순 공지 요약이 아니라 구조, 수혜자, 작동 방식, 한계 중 최소 두 축을 선명하게 풀었는가
- structure: 리드, 배경, 솔루션, 과제, 전망 흐름이 잡히는가
- structure: 배경:, 과제:, 전망:, 해석: 같은 소제목이나 라벨을 본문에 노출하지 않는가
- structure: 본문이 4~5문단 중심으로 정리되어 있는가
- flow: 문단 전환과 반복이 자연스러운가
- flow: 쉼표를 남발하지 않고 문장 하나에 아이디어 하나씩 담는 흐름인가
- conciseness: 군더더기 없이 짧고 밀도 있게 썼는가
- conciseness: 한 문장에 쉼표가 2개 이상 반복되지 않는가
- conciseness: 정보축이 적은 기사라도 조치, 대상, 작동 방식, 조건이 완결되어 있으면 짧은 분량만으로 과소평가하지 않는가
- conciseness: 분량보다 완결성이 우선이라는 원칙을 지키되, 짧은 기사도 최소 3개 정보축을 담는가
- conciseness: 분량이 지나치게 짧아 핵심 구조가 사라지지 않았는가
- naturalness: 번역투, 어색한 조사, 주어 증발이 없는가
- accuracy: 수치, 고유명사, 원인-결과가 일관되는가
- coherence: 제목이 약속한 앵글을 끝까지 유지하는가
- tone_headline: 평어체, 제목 30자 이내, 쉼표 없음, 과장 없음
- tone_headline: 보도자료식 찬사나 자기홍보 문장이 아닌가

출력 규칙
- 통과 시 JSON만 출력한다.
- 미통과 시 JSON을 먼저 출력하고, `---` 뒤에 수정 기사를 출력한다.
- JSON 형식은 반드시 `scores`, `total`, `pass`, `fails`를 포함한다.
- 수정 기사도 평어체, 순수 텍스트, 완결된 상태로 쓴다.
- 통과 기준은 total >= 72다.
