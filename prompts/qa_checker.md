아래 기사를 검수하고 출력 규칙에 따라 출력한다. 통과 시 JSON만, 미통과 시 JSON + --- + 수정기사 순서로 출력한다.

[매체 특성] {media_tone}

즉시 탈락 조건 (해당 시 무조건 탈락 + 수정 기사 출력)
- 제목 규칙 위반: 제목에 쉼표(,) 포함 또는 30자 초과
- 내부 추론 노출: Step, Phase, "생각:", "검토:", "분석:" 같은 사고과정 라벨 포함
- 미완성 기사: 문장이나 단락이 중간에 끊김, 말줄임표로 종료, 결론 없이 종료
- 포맷 오염: 마크다운(##, **, `), HTML 태그 포함 (제목:, 리드문:, 본문:, 카테고리:, 태그: 라벨은 허용)
- 사실 위반: 원문에 없는 수치, 고유명사, 사건을 사실처럼 단정
- 시제 승격: 계획, 검토, 예정, 추진 중을 시행, 완료, 확정으로 바꿈
- 분량 미달: 본문이 원문 분량의 80% 미만인 경우 (원문 분량: {source_chars}자 → 기준 {min_chars}자)

채점 항목
- lead: 첫 2문장 안에 핵심과 수혜자, 해결책, 임팩트가 있는가
- structure: 리드, 배경, 솔루션, 과제, 전망 흐름이 잡히는가
- structure: 배경:, 과제:, 전망: 같은 소제목이나 라벨을 본문에 노출하지 않는가
- structure: 본문이 4~5문단 중심으로 정리되어 있는가
- flow: 문단 전환과 반복이 자연스러운가
- flow: 쉼표를 남발하지 않고 문장 하나에 아이디어 하나씩 담는 흐름인가
- conciseness: 군더더기 없이 짧고 밀도 있게 썼는가
- conciseness: 한 문장에 쉼표가 2개 이상 반복되지 않는가
- conciseness: 본문이 1200자 이상으로 충분한 밀도를 갖추는가
- naturalness: 번역투, 어색한 조사, 주어 증발이 없는가
- accuracy: 수치, 고유명사, 원인-결과가 일관되는가
- coherence: 제목이 약속한 앵글을 끝까지 유지하는가
- tone_headline: 평어체, 제목 30자 이내, 쉼표 없음, 과장 없음

출력 규칙
- 통과 시: JSON만 출력한다.
- 미통과 시: 반드시 아래 순서로 출력한다.
  1. JSON 출력
  2. --- (구분자)
  3. 수정 기사 전문 출력 (제목: / 리드문: / 본문: / 카테고리: / 태그: 형식)
- JSON만 출력하고 수정 기사를 생략하는 것은 절대 금지한다.
- 분량 미달로 탈락한 경우 수정 기사 본문은 원문 팩트 범위 안에서 각 문단의 배경·맥락·의미를 더 풀어 서술하여 반드시 {min_chars}자 이상으로 완성한다. 없는 수치나 사실을 만들지 않는다.
- JSON 형식: scores, total, pass, fails 포함.
- 수정 기사: 평어체, 순수 텍스트, 완결 상태.
- 통과 기준: total >= 72.
